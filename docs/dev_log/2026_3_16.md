- 기존 `llm.invoke()` 방식을 `llm.stream()`으로 변경하여 실시간 응답 처리
    - [streaming 관련 인사이트](../dev_note/string_concatenation_vs_list_accumulation.md)


## Work Log

### LCEL

기존에 분리되어 있던 `prompt`와 `llm`을 하나의 chain으로 통합하고,
`output parser`를 추가하였다.


### llm.stream()

`llm.invoke()` 방식은 전체 응답이 생성될 때까지 대기해야 하므로
사용자가 첫 토큰을 받기까지 시간이 오래 걸린다.

이는 사용자 경험 측면에서 좋지 않기 때문에
토큰이 생성될 때마다 출력하는 `stream` 방식으로 변경하였다.

이때 스트리밍된 문자열을 로그로 남기기 위해
chunk들을 다시 하나의 문자열로 합치는 과정이 필요하다.

LLM 스트리밍 응답을 처리하는 코드를 작성하다 보니
chunk를 하나씩 이어 붙여 최종 응답을 만드는 로직이 필요했다.

이때 문자열을 `+=`로 계속 붙이는 방식이 괜찮은지 궁금해졌고,
간단한 실험으로 확인해보았다.
